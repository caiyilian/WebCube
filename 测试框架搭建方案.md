# WebCube 测试框架搭建方案

> 生成日期：2026-06-28 | 当前阶段：**Phase 1-4 全部完成** | 目标：全模块覆盖（客户端 + 服务端）
> **⚠️ 交互模型已变更**：鼠标拖拽仅旋转视角（Orbit），不再直接旋转魔方面。魔方转动需先点击选中某个正方形，再用 WASD/方向键操作。

---

## 一、交互模型说明（重要变更）

### 1.1 新交互模型

```
┌─────────────────────────────────────────────────────────────┐
│  步骤 1：鼠标拖拽 → 仅旋转视角（Orbit Camera）                │
│          不会触发任何魔方转动                                  │
│                                                             │
│  步骤 2：鼠标点击 → 选中魔方某个面的某个正方形                 │
│          （3×3 魔方，6 个面，每面 9 个正方形，共 54 个）       │
│          选中后该正方形高亮显示                                │
│                                                             │
│  步骤 3：键盘操作 → WASD / 方向键 → 旋转选中的正方形所在面     │
│          每个正方形只有 4 个操作方向：上/下/左/右              │
│          例如：选中右面(R)中心格 → W=顺时针 / S=逆时针         │
│                  选中上面(U)中心格 → A=左旋 / D=右旋          │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 与旧模型的差异

| 操作 | 旧模型 | 新模型 |
|------|--------|--------|
| 鼠标拖拽魔方 | 直接旋转对应面 | **仅旋转视角（Orbit）** |
| 键盘 R/L/U/D/F/B | 直接旋转对应面 | **不再支持** |
| 点击魔方 | 无效果 | **选中某个正方形** |
| WASD / 方向键 | 无效果 | **旋转选中正方形所在面** |

### 1.3 对测试的影响

| 测试文件 | 影响 | 说明 |
|---------|------|------|
| `Interaction.test.ts` | **重写** | 拖拽旋转逻辑删除，改为点击选中 + WASD 旋转 |
| `keyboard-ops.spec.ts` | **重写** | R/L/U/D/F/B 测试删除，改为 WASD/方向键 + 点击选中 |
| `hud-buttons.spec.ts` | 不变 | HUD 按钮功能不变 |
| `cube-rendering.spec.ts` | 微调 | 需验证选中高亮效果 |
| `useGameStore.test.ts` | 不变 | 状态机逻辑不变 |

## 三、依赖安装

### 3.1 客户端测试 — Vitest

```bash
npm install -D vitest @vitest/coverage-v8 jsdom
```

- **Vitest**：与 Vite 原生集成，启动快，API 与 Jest 兼容
- **@vitest/coverage-v8**：代码覆盖率报告
- **jsdom**：模拟浏览器 DOM 环境，用于组件测试

### 3.2 E2E 测试 — Playwright

```bash
npm install -D @playwright/test
```

- **@playwright/test**：浏览器自动化测试框架
- 支持 Chromium / Firefox / WebKit 三浏览器
- 内置截图、视频录制、trace 录制

### 3.3 服务端测试 — Vitest（复用）

```bash
# 复用已有的 Vitest，无需额外安装
```

### 3.4 安装命令（一键）

```bash
npm install -D vitest @vitest/coverage-v8 jsdom @playwright/test
```

## 九、注意事项

### 9.1 代码问题（测试前需修复）

| 问题 | 文件 | 说明 |
|------|------|------|
| HUD 构造函数签名不一致 | `HUD.tsx` vs `App.tsx` | `HUD.tsx` 无参，`App.tsx` 传 `'practice'` |
| HUD 缺少方法 | `HUD.tsx` | `setTimerDisplay`, `setHintActive`, `setCallbacks` 未定义 |
| `@shared/types.js` 路径 | 多处 | tsconfig 中 `@shared` 映射到 `shared/`，但 import 用 `.js` 扩展名 |
| `CubeRenderer.updateFromState` | `CubeRenderer.ts` | 方法体为空，仅 `void` 占位 |
| `RoomManager.applyScramble` | `server/rooms/RoomManager.ts` | 方法体为空，返回还原状态 |
| `RoomManager.applyMoveToState` | `server/rooms/RoomManager.ts` | 方法体为空，返回原状态 |
| `server/index.ts` 使用 `@ts-nocheck` | `server/index.ts` | 服务端类型检查关闭 |

### 9.2 测试策略调整

- **单元测试先行**：纯逻辑测试写起来快，反馈快，能覆盖核心业务
- **E2E 测试后置**：需要 dev server 运行，启动慢，适合集成验证
- **截图基准图**：首次运行 E2E 时用 `--update-snapshots` 生成，之后对比检测回归
- **服务端测试**：RoomManager/Matchmaking/TournamentManager 可独立测试，无需启动 Socket.IO
- **多人联机 E2E**：需同时启动前端和后端，可考虑用 `playwright.config.ts` 的 `webServer` 配置启动两个服务
- **CI 集成**：后续可接入 GitHub Actions，每次 PR 自动跑测试
